# 施設点数管理

居宅療養管理指導費・在宅患者訪問薬剤管理指導料の区分を自動計算する、調剤薬局向けの補助ツールです。施設別の患者一覧表をデジタル化し、月ごとの訪問実績を記録して翌月請求前のチェックに使います。

> このアプリは請求前チェック用の補助ツールです。最終請求はレセコンと制度確認を前提にしてください。

## 主な機能

- 月別の訪問登録（タップ1回でON/OFF、タップ日が訪問日）
- 区分の自動判定（518/379/342 単位、650/320/290 点）
- 特例対応（10%以下、20戸未満2人以下、グループホームのユニット別判定、個人宅同一世帯）
- 棟・ユニット間および施設間の移動履歴管理（訪問日の在籍棟で集計）
- 入退院・終了・再開などのイベント記録
- 計算結果の判定根拠を文章で表示
- JSONバックアップ／復元、CSV出力、印刷
- ダミーデータで初期実装、実在患者名は不使用
- 患者データはブラウザ内（IndexedDB）にのみ保存、外部送信なし
- iPad PWA 対応（ホーム画面に追加でオフライン動作）

## 開発

### 必要環境

- Node.js 20 以上
- npm

### セットアップ

```bash
npm install
npm run dev
```

開発サーバーが `http://localhost:5173/shisetsu-tensu/` で起動します。

### テスト

```bash
npm test
```

### ビルド

```bash
npm run build
```

`dist/` フォルダに静的ファイルが生成されます。

2. リポジトリ設定 → **Pages** → **Source** を **GitHub Actions** に変更

git branch -M main

## iPad での使い方

1. iPad の Safari で公開URLを開く
2. 共有ボタン → **「ホーム画面に追加」**
3. ホーム画面のアイコンから起動

以降は通常のアプリのように使えます。インターネット接続がなくても動作しますが、初回ロードと更新の取得時のみオンラインが必要です。

## バックアップ運用（重要）

患者データはブラウザ内（IndexedDB）にのみ保存されています。**Safariのデータ消去・iPadの初期化・PWAの削除と再インストール** などでデータが消える可能性があるため、以下の運用を推奨します。

### 月初の請求作業後

1. アプリの **設定画面** → **「JSONエクスポート」** をタップ
2. 共有メニュー → **「ファイルに保存」** → **iCloud Drive** に保存
3. ファイル名は `backup_YYYY-MM-DD.json`（自動）
4. 古いバックアップも残しておく（複数世代）

### 復元するとき

1. 設定画面 → **「JSONインポート」**
2. iCloud Drive から該当ファイルを選択
3. 確認ダイアログ → **続行**

## 計算ロジック

詳細はソースコード [`src/domain/calc.ts`](src/domain/calc.ts) を参照してください。要約すると以下の順序で判定します。

1. 算定対象月を基準に、訪問実績がある患者のみを計算対象とする
2. 介護＋介護予防は同じ建物内で合算して人数判定
3. 医療保険は介護とは別に人数判定
4. 通常施設では同じ建物内の算定対象人数で判定（518/379/342 または 650/320/290）
5. グループホームでユニット数3以下の場合は、ユニットごとの人数で判定（自動）
6. その他の施設種別でも、棟ごとに「別建物として算定」をONにすればユニット別判定が可能
7. 戸数の10%以下なら各患者を1人区分（518/650）として算定
8. 戸数20戸未満かつ算定対象2人以下なら各患者を1人区分として算定
9. 個人宅で同一世帯の複数患者がいる場合は各患者を1人区分として算定
10. 棟・施設移動があった場合は訪問日時点の在籍棟で集計
11. 計算根拠を文章で必ず表示

## ライセンス

社内利用を想定しており、外部公開はしていません。

## 注意事項

- 実在患者名は登録時にも使用しないでください（ダミー名・イニシャル等で運用）
- 患者データは外部APIに送信されません
- 単位数・点数は制度改定時に **設定画面** から編集してください
